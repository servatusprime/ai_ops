# Guide: Alignment Notes

## Purpose

Keep short alignment notes when guidance overlaps policy or spec requirements.

## Required Elements

- affected artifact paths
- conflict/overlap summary
- resolved direction
- owner and date

## Canonical Pattern

```markdown
> Alignment Note: This guide defers to `00_Admin/policies/<policy_name>.md` for enforcement.
```

## Usage Rules

- Use notes for concise cross-artifact clarity.
- Promote persistent alignment decisions to canonical policy/spec updates.
